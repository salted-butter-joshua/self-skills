# Web Page Knowledge Extract

从网页链接提取结构、分块总结、生成记忆卡片。全程以页面原文为唯一事实来源，禁止臆造。

## 触发与输入

- 输入：一个或多个 HTTP(S) URL；可选粘贴的网页正文（当抓取失败时）
- 输出：Markdown 报告（总览 + 分主题总结 + 记忆卡片），保存到用户指定目录；未指定时默认 `~/Documents/notes/web-knowledge/`

## 工具优先级

1. **WebFetch** — 首选，获取可读 Markdown/文本
2. **Shell + curl** — WebFetch 失败或内容过短时重试（带 User-Agent）
3. **用户粘贴** — 两次抓取仍失败时，请用户粘贴正文并标注来源 URL

抓取后保留：页面标题、URL、抓取时间（`date`）、可见的章节标题层级。

## 工作流

复制并跟踪进度：

```
- [ ] 1. 抓取与清洗
- [ ] 2. 结构总览（主题 / 子主题）
- [ ] 3. 分块规划（窗口安全）
- [ ] 4. 分块精读与总结
- [ ] 5. 记忆卡片生成
- [ ] 6. 全文汇编与自检
```

### Step 1：抓取与清洗

1. 用 WebFetch 获取页面；记录原始长度（字符数）
2. 清洗：去导航/footer/广告重复块；保留正文标题与段落
3. 若正文 &lt; 200 字或明显为登录墙/验证码页 → 报告问题，不继续编造

### Step 2：结构总览

根据 **标题层级（h1–h6）** 与 **段落换行** 构建大纲：

```markdown
## 页面总览

| 字段 | 内容 |
|------|------|
| 标题 | {页面 title} |
| URL | {url} |
| 主题（一句话） | {整页在讲什么} |
| 子主题数 | {n} |

### 大纲树

- H1/H2/H3 ... 标题
  - 预估字数
  - 子主题核心（≤ 40 字，仅概括该节可见内容）
```

**子主题核心**：用原文可支持的短语，不写推测。

### Step 3：分块规划（窗口安全）

目标：每块送入 LLM 精读时不超过上下文安全上限。

| 语言倾向 | 单块上限（字符） | 说明 |
|----------|------------------|------|
| 中文为主 | 6,000 | 约 3k–4k tokens |
| 英文为主 | 12,000 | 约 3k tokens |
| 混合 | 8,000 | 取保守值 |

**分块规则**（按优先级）：

1. **优先按标题切** — 不在段落中间切断同一论点
2. **小节合并** — 单节 &lt; 400 字且与相邻节同主题 → 合并为一块
3. **大节拆分** — 单节超上限 → 按 h3/h4 或空行段落组再切
4. **无标题页** — 按每 ~5000 字在段落边界切，块首生成临时小标题（标注「推断结构」）

输出分块表：

```markdown
## 分块计划

| 块 ID | 覆盖标题 | 字符数 | 备注 |
|-------|----------|--------|------|
| B1 | ... | ... | |
```

块数应使每块总结 + 卡片生成后，全文汇编仍低于模型窗口（留 ≥30% 给对话与输出）。

### Step 4：分块精读与总结

对 **每一块** 单独处理（可并行多块，但每块独立读原文片段）：

**铁律**

- 只写原文中出现或可严格推导的信息
- 数字、专名、日期、公式与原文一致
- 原文未写清时写「原文未说明」，不补全
- 每个要点可追溯：括号注明来源标题，如 `（见 §安装步骤）`

**每块输出模板** — 对该块内每个 **主要标题（h2 或块内最高级标题）** 各写一节：

```markdown
### [{标题名}]

**是什么**  
{定义/现象/结论——原文说了什么}

**为什么**  
{原因/背景/动机——原文有则写，无则写「原文未说明」}

**怎么做**  
{步骤/方法/实践——原文有则写；列表逐步；无则写「原文未说明」}

**关键事实（可追溯）**  
- {事实 1}（依据：…）
- {事实 2}（依据：…）

**本块未覆盖**  
{若用户关心的点在本块无信息，列出以免误以为全页没有}
```

不要为凑三段而编造「为什么/怎么做」。

### Step 5：记忆卡片生成

基于 Step 4 的总结，为每个主要子主题生成 **3–8 张** 卡片（内容少则减少）。

卡片类型（每张只选一种主类型）：

| 类型 | 正面 | 背面 |
|------|------|------|
| 概念 | {术语/概念} 是什么？ | 是什么 + 一句依据 |
| 因果 | 为什么 {现象}？ | 原因要点 + 依据 |
| 方法 | 如何 {任务}？ | 步骤要点 + 依据 |
| 事实 | {可考察的事实问句} | 答案 + 依据 |

**卡片铁律**

- 背面答案必须能在原文找到对应句或段落
- 不考原文没写的细节
- 背面末尾统一：`依据：{章节名或原文短引 ≤20 字}`

输出格式见 [reference.md](reference.md#记忆卡片格式)。

### Step 6：全文汇编与自检

合并为一份文件：

```markdown
# {页面标题} — 知识提取

> 来源：{URL}  
> 抓取时间：{ISO 时间}

## 一、页面总览
{Step 2}

## 二、分块计划
{Step 3}

## 三、分主题知识总结
{Step 4 全部块，按阅读顺序}

## 四、记忆卡片
{Step 5 全部卡片，按子主题分组}

## 五、自检清单
- [ ] 所有「是什么/为什么/怎么做」均有原文依据或标注未说明
- [ ] 无臆造专名、数据、结论
- [ ] 分块未超字符上限
- [ ] 卡片数与子主题复杂度匹配
```

**保存**：`{YYYYMMDD}--web-knowledge-{简短标题}.md` 到输出目录，并告知用户完整路径。

## 与 ljg-card 协作

用户要求「做成图/铸/视觉卡片」时：先完成本 skill 的 Markdown 输出，再将 **第三节总结 + 第四节卡片** 作为输入，调用 `ljg-card`（默认 `-l` 或用户指定模具）。

## 失败与边界

| 情况 | 处理 |
|------|------|
| 动态渲染 / SPA 空白 | 说明需浏览器渲染；请用户粘贴正文或提供 PDF/截图 OCR 文本 |
| 付费墙 / 登录 | 不编造；请用户提供可访问副本 |
| 超长文档（&gt;50 块） | 先输出总览 + 前 N 块，询问是否继续其余块 |
| 非文章页（首页/列表） | 在总览中说明页面类型，只对正文条目逐条处理或请用户指定子链接 |
