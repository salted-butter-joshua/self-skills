# Web Page Knowledge — Reference

## 字符估算（快速）

| 内容 | 估算 tokens |
|------|-------------|
| 1 个中文字符 | ~0.5–1 |
| 1 个英文单词 | ~1 |
| 输出预留 | 每块总结+卡片约占输入 30–50% |

**安全公式**：`块字符数 × 1.5 + 预计输出 2000` &lt; 模型可用窗口 × 0.6

常见模型单轮可用输入约 100k–128k tokens 时，中文块 6000 字仍安全；上下文较小时（如 32k）将单块降至 **3000 中文字**。

## 标题识别优先级

1. Markdown 标题行 `#` … `######`
2. HTML 转换后的 `##` 层级
3. 加粗短行 + 独立成段（弱标题，标注「弱结构」）
4. 无结构 → 按字数切块 + 临时标题

## 记忆卡片格式

### Markdown（默认）

```markdown
## 记忆卡片 · {子主题名}

### 卡片 1 · 概念
**Q：** {正面问句}  
**A：** {背面答案}  
**依据：** {章节名 / 原文短引}

### 卡片 2 · 方法
...
```

### JSON（可选，便于 Anki/导入）

```json
{
  "source_url": "https://...",
  "section": "子主题名",
  "cards": [
    {
      "type": "concept",
      "front": "问句",
      "back": "答案",
      "evidence": "章节或原文摘录"
    }
  ]
}
```

用户未要求 JSON 时只输出 Markdown。

## 卡片质量检查

每张卡片过以下筛子，不通过则改写或删除：

1. **可答性** — 正面是明确问题，背面能独立理解
2. **可证性** — 背面每一句在原文有对应
3. **非琐碎** — 不考「本页第几段」类元信息
4. **非重复** — 同一子主题内不重复考同一点

## 输出样例（节选）

```markdown
### [安装环境]

**是什么**  
原文说明需 Python 3.10+ 与 Node 18，用于运行本地开发服务器。

**为什么**  
原文未说明。

**怎么做**  
1. 安装 Python 3.10+（见 §安装环境）  
2. `pip install -r requirements.txt`  
3. `npm install && npm run dev`

**关键事实（可追溯）**  
- 最低 Python 3.10（依据：§安装环境）

## 记忆卡片 · 安装环境

### 卡片 1 · 方法
**Q：** 按原文，本地开发需要先做什么？  
**A：** 安装 Python 3.10+，再 pip 安装依赖，最后 npm install 并 npm run dev。  
**依据：** §安装环境
```

## 反幻觉用语表

| 场景 | 写法 |
|------|------|
| 原文没写原因 | 原文未说明 |
| 原文没写步骤 | 原文未说明 |
| 推测但不确定 | 不使用；改为「原文未说明」 |
| 多处矛盾 | 并列列出并标注各自依据章节 |
